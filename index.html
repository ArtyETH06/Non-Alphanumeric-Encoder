Non-Alphanumeric-Decoder (NAD)

Demo — bypassing a PHP validation using non-alphanumeric XOR

Exploit Encoder Decoder

Exploit — step by step

  1. Find the vulnerable code snippet: 
    if(!preg_match('/[a-zA-Z`]/', $_POST['input'])){ eval('print '.$_POST['input'].";"); }
  2. Understand the restriction: letters and the backtick ` are blocked by the regex.
  3. Use PHP's ^ operator (bitwise XOR) between two characters to produce a letter output (e.g. "#" ^ "@" may produce c depending on ASCII codes).
  4. Concatenate multiple XOR expressions using PHP's concatenation operator . to form a full string (e.g. ("#"^"@").("!"^"@") ...).
  5. Submit the payload: the regex passes (no letters), then eval executes the reconstructed expression.

This is a pedagogical demo. Do not test this technique on systems you don't own or have permission to test.

Tip

Multiple pairs of non-alphanumeric characters can XOR to the same letter. The generator picks the first printable non-alphanumeric pair found for each character.
